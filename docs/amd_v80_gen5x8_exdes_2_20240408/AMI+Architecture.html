AMI Architecture¶

Overview¶

The AVED Management Interface (AMI) provides host-access to control and manage the AVED Management Controller (AMC).

It consists of:

  • A kernel driver, providing access to the AVED platform.

  • An API, providing a user-space abstraction to the kernel driver.

  • A command line tool (ami_tool), providing a human-readable interface to the API.

Note: The AMI is an example design demonstrating how an AVED host might look and be developed. It is not intended to be the final word in AVED run time management and control.
